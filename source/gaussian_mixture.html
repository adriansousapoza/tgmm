Bases: Module

A Gaussian Mixture Model (GMM) based on Expectation-Maximisation (EM) implemented in PyTorch.

This GMM supports:

  • The Expectation-Maximization (EM) algorithm.

  • The Classification EM (CEM) algorithm as an alternative to standard EM.

  • Multiple initializations (n_init).

  • Configurable covariance types (full, diag, spherical, tied_full, tied_diag, tied_spherical).

  • Maximum Likelihood Estimation (MLE) and Maximum a Posteriori (MAP) estimation (i.e. with Priors).

Parameters:

  • n_features (int) ‚Äì Dimensionality of the input data (number of features).

  • n_components (int, optional) ‚Äì Number of mixture components (default: 1).

  • covariance_type (str, optional) ‚Äì

    Type of covariance parameters to use. Must be one of: ‚Äòfull‚Äô, ‚Äòdiag‚Äô, ‚Äòspherical‚Äô, ‚Äòisotropic‚Äô, ‚Äòtied_full‚Äô, ‚Äòtied_diag‚Äô, ‚Äòtied_spherical‚Äô.

    Note:

    • ‚Äôisotropic‚Äô is an alias for ‚Äòspherical‚Äô

    • ‚Äôfull‚Äô is an alias for ‚Äòtied_full‚Äô.

    (default: ‚Äòfull‚Äô)

  • tol (float, optional) ‚Äì Convergence threshold for EM (relative improvement in log-likelihood). (default: 1e-4)

  • reg_covar (float, optional) ‚Äì Non-negative regularization added to the diagonal of covariance. Helps keep covariance matrices from becoming singular. (default: 1e-6)

  • max_iter (int, optional) ‚Äì Maximum number of EM iterations to perform. (default: 1000)

  • init_params (str, optional) ‚Äì Method for initializing means (kmeans, random, points, kpp, maxdist). (default: ‚Äòkmeans‚Äô)

  • weights_init (torch.Tensor or None, optional) ‚Äì User-provided initial component weights of shape (n_components,). If None, weights are set uniformly. (default: None)

  • means_init (torch.Tensor or None, optional) ‚Äì User-provided initial component means of shape (n_components, n_features). If None, means are randomized according to init_params. (default: None)

  • covariances_init (torch.Tensor or None, optional) ‚Äì User-provided initial covariances. Shape depends on covariance_type. (default: None)

  • n_init (int, optional) ‚Äì Number of random initializations to try. The best run (highest log-likelihood) is kept. When n_init > 1, each initialization uses a different but deterministic random state derived from random_state to ensure both reproducibility and diversity. Specifically, initialization i uses random_state + i as its seed. (default: 1)

  • random_state (int or None, optional) ‚Äì Random seed for reproducibility. If None, uses PyTorch‚Äôs internal seed. When n_init > 1, this serves as the base seed, with each initialization using random_state + initialization_index to ensure different but reproducible initializations. (default: None)

  • warm_start (bool, optional) ‚Äì If True, reuse the solution of the previous call to fit as initialization. (default: False)

  • verbose (bool, optional) ‚Äì If True, print progress during EM iterations. (default: False)

  • verbose_interval (int, optional) ‚Äì Frequency (in iterations) at which to print progress. (default: 10)

  • device (str or None, optional) ‚Äì Device on which to run computations (‚Äòcpu‚Äô or ‚Äòcuda‚Äô). If None, uses GPU if available, otherwise CPU. (default: None)

  • weight_concentration_prior (torch.Tensor or None, optional) ‚Äì Dirichlet concentration prior for the mixture weights, used in MAP. (default: None)

  • mean_prior (torch.Tensor or None, optional) ‚Äì Prior for the means, used in MAP. If provided, must match mean_precision_prior. (default: None)

  • mean_precision_prior (float or None, optional) ‚Äì Precision of the mean prior, used in MAP. (default: None)

  • covariance_prior (torch.Tensor or None, optional) ‚Äì Prior on covariance(s), used in MAP. If provided, must match degrees_of_freedom_prior. (default: None)

  • degrees_of_freedom_prior (float or None, optional) ‚Äì Degrees of freedom in the Wishart prior for covariances (MAP). (default: None)

  • cov_init_method (str, optional) ‚Äì Method for initializing covariances if covariances_init is None. Supported: ‚Äòeye‚Äô, ‚Äòrandom‚Äô, ‚Äòempirical‚Äô. (default: ‚Äòeye‚Äô)

  • cem (bool, optional) ‚Äì If True, use the Classification EM algorithm instead of standard EM. (default: False)

Variables:

  • weights (torch.Tensor) ‚Äì Mixture component weights of shape (n_components,).

  • means (torch.Tensor) ‚Äì Mixture component means of shape (n_components, n_features).

  • covariances (torch.Tensor) ‚Äì Mixture component covariances. Shape depends on covariance_type.

  • initial_weights (torch.Tensor) ‚Äì Initial mixture component weights before EM optimization of shape (n_components,). These are the weights after initialization but before any EM iterations.

  • initial_means (torch.Tensor) ‚Äì Initial mixture component means before EM optimization of shape (n_components, n_features). These are the means after initialization but before any EM iterations.

  • initial_covariances (torch.Tensor) ‚Äì Initial mixture component covariances before EM optimization. Shape depends on covariance_type. These are the covariances after initialization but before any EM iterations.

  • fitted (bool) ‚Äì Whether the model has been fitted. This attribute is important when using warm_start.

  • converged (bool) ‚Äì Whether the EM algorithm has converged in the best run.

  • n_iter (int) ‚Äì Number of EM iterations performed in the best run.

  • lower_bound (float) ‚Äì Log-likelihood lower bound on the fitted data for the best run.

  • best_random_state (int or None) ‚Äì The random state that produced the best result when n_init > 1. This is useful for reproducing the specific best initialization. Only available after fitting when random_state is not None and n_init > 1.
